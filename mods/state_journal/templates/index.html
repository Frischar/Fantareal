生成规则worker

这里决定辅助模型怎么读取最近聊天、什么时候写入心笺、是否生成主角状态卡，以及允许用户附加哪些安全提示。

轮数越高，worker 读取的最近聊天越多；建议默认 3 轮，剧情复杂时可提高到 5。

主角状态卡安全生成

主角状态卡只写可观察状态、身体状态、伤势、气息、位置与互动关系；不会替用户决定行动、台词或内心。

安全提示词开放仅附加

这里只会附加到安全区域，不会开放 JSON 根结构、updates 协议、SQLite 写入规则或字段 key。

幕笺显示

控制 Chat 里是否显示幕笺、使用哪个模板和外观包。

模板决定角色状态卡生成哪些字段；美化包决定标题卡、角色卡和关系卡的外观。恢复默认会启用 Chat 幕笺，并切回“标准状态模板 + 标准样式”。

折叠适合沉浸阅读；默认展开适合状态栏玩家；简洁状态条只占一行；不显示仅保留后台记录。

生成风格影响什么？

标题生成风格只控制本轮标题卡；附笺生成风格控制角色状态卡。外观样式仍由卡片模板负责。

附笺会先生成完整数据，再由前端筛选显示。Chat 状态显示模式只影响聊天页展示，不改变 SQLite 后台记录。

幕笺模板

模板开放第一版：控制角色附笺“生成哪些字段、每个字段怎么写、最终按什么格式展示”。暂不开放 HTML/CSS/JS 外观。

字段列表
