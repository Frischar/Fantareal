# 如何制作心笺美化包

本文说明如何制作心笺“幕笺外观模板包”。

## 1. 美化包是什么

美化包决定幕笺在 Chat 页面如何显示。它可以改变：

```text
颜色
布局
卡片结构
字段分组
字段显示名
图标
数值显示方式
图片资源
```

但美化包不能决定辅助模型生成哪些字段。

## 2. 美化包和模板的区别

```text
模板：决定输出什么内容。
美化包：决定内容怎么显示。
```

美化包不能删字段。用户模板里有多少字段，Chat 幕笺就应该尽量完整显示多少字段。

## 3. 基础结构

一个美化包通常包含：

```json
{
  "id": "my_theme",
  "name": "我的美化包",
  "version": "1.0.0",
  "author": "作者名",
  "description": "这个美化包的用途",
  "style": {
    "class_name": "theme-my-theme",
    "accent": "#c98263",
    "layout": {
      "layout_type": "paper_time",
      "character_card": "paper_rows"
    },
    "tokens": {
      "emotion": "情",
      "clothing": "衣"
    }
  }
}
```

## 4. layout_type 怎么选

当前推荐三类特色布局：

```text
status_panel_pro  状态面板 Pro：HUD、进度条、多层卡片
storyboard        剧本分镜：CUT、镜头组、表演组、细节组
paper_time        纸页时间卡：手札、时间笺、旁注长文本
```

普通样式可以不写 `layout_type`，走默认字段卡。

## 5. labels / tokens 的作用

`tokens` 可以给字段换显示名或图标。

示例：

```json
"tokens": {
  "emotion": "情绪",
  "visual_focus": "视线",
  "favor_level": "FAVOR"
}
```

这只改变显示，不改变字段本身。

## 6. progress_bars 的作用

状态面板 Pro 可以把标准数值字段显示成进度条。

示例：

```json
"progress_bars": [
  { "key": "favor_level", "label": "FAVOR", "name": "好感", "max": 100 },
  { "key": "trust_level", "label": "TRUST", "name": "信任", "max": 100 },
  { "key": "guard_level", "label": "GUARD", "name": "戒备", "max": 100 },
  { "key": "pulse_level", "label": "PULSE", "name": "心绪", "max": 100 }
]
```

普通样式会显示成：

```text
好感：65/100（+2）
```

状态面板 Pro 会显示成：

```text
FAVOR  ██████░░░░ 65  ↑ +2
```

## 7. 图片资源怎么写

推荐两种方式：

```text
1. 内嵌 data URI：适合随 JSON 一起导入
2. static 路径：适合插件内置资源
```

内嵌示例：

```json
"media": {
  "scene_image_data": "data:image/webp;base64,..."
}
```

## 8. 导入、切换、删除

在心笺设置页：

```text
幕笺外观模板包 → 导入 JSON
幕笺外观模板包 → 选择美化包
幕笺外观模板包 → 删除导入包
```

内置包不可删除，导入包可以删除。

## 9. 常见错误

### JSON 导入失败

检查是否是合法 JSON，逗号、引号和大括号是否完整。

### layout_type 不生效

检查 `style.layout.layout_type` 是否写对。

### 图片不显示

检查图片路径或 data URI 是否存在。新版心笺会保留 `media` 字段。

### 字段变少

这是错误行为。新版规则是：美化包不能删字段，只能改变显示方式。

## 10. 制作建议

```text
状态面板 Pro：适合数值、HUD、状态栏。
剧本分镜：适合演出感、镜头感、插图。
纸页时间卡：适合古风、长文本、旁注阅读。
```
