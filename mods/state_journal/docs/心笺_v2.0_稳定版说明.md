# 心笺 v2.0.0-state-journal-stable

本版本为心笺稳定整合版，主要用于封版，不继续扩展新功能。

## 版本定位

心笺在 v2.0.0 进入稳定维护阶段。该版本整合了此前 v1.18 与 v1.19 系列的核心能力：角色配置、变量阶段、世界书阶段标签、外观包管理、幕笺显示与 worker 状态兜底。

## 核心能力

### 角色配置

- 支持多角色卡：主卡可作为旁白或总设定，多角色 personas 作为心笺角色。
- 支持单角色卡：没有多角色时，可将主卡作为唯一心笺角色。
- 支持角色来源识别：自动识别 / 主卡就是角色 / 主卡是旁白，只读取多角色。
- 支持变量、阶段规则、状态快照字段配置。
- 支持“同步至角色卡”确认弹窗，明确只同步配置，不同步运行数值或历史。

### 世界书联动

- 支持心笺阶段标签 external_tag。
- 标签格式固定为：`state_journal.stage.<role_id>.<stage_key>`。
- 支持从心笺阶段创建世界书条目。
- 世界书条目可显示绑定状态和当前命中状态。
- 心笺负责阶段判断，世界书负责阶段表现。

### 外观模板与美化包

- 支持内置外观包与外置美化包。
- 支持导入、导出、删除和保存。
- 外置包删除有确认弹窗。
- 删除当前使用的外置包时自动回退到标准样式。
- 标准样式跟随主程序浅色 / 暗色 / 透明度与毛玻璃效果。
- HUD / 状态面板类外观包保留进度条特色。
- 自定义变量默认显示中文变量名，并支持进度条兜底。

### 幕笺显示与运行稳定性

- 支持标题卡、场景信息、角色状态、本轮数值变化、关系变化与阶段变化。
- 支持 reroll 后清理旧任务提示。
- worker 开始后会写入 running 状态。
- worker 异常、超时、解析失败或保存失败会写入错误状态。
- Chat 前台支持生成较慢提示和重试入口。

## v2.0.0 本次改动

- 前端缓存标记统一为 `state-journal-v200-state-journal-stable`。
